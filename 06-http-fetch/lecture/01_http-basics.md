# 6주차 - HTTP + fetch 1: HTTP 기초

## 이번 주 목표

- HTTP가 무엇인지 설명할 수 있다.
- Request와 Response의 차이를 이해할 수 있다.
- API 요청이 왜 필요한지 알 수 있다.

## 왜 이 실습을 하나요?

현대 웹페이지는 정적인 문서만 보여주지 않고 외부 서버에서 데이터를 가져옵니다.
게시글 목록, 날씨 정보, 사용자 정보처럼 다양한 데이터가 HTTP 요청을 통해 화면에 들어옵니다.

## 준비물

- 브라우저
- `06-http-fetch/example/public-api-viewer/` 예제
- 공개 API 주소 `https://koreandummyjson.vercel.app`

## 이번 주 파일 설명

- `06-http-fetch/example/public-api-viewer/index.html`: 버튼과 결과 영역이 있는 화면입니다.
- `06-http-fetch/example/public-api-viewer/script.js`: API 요청을 보내는 코드입니다.

## 코드에서 꼭 볼 부분

- `Request`: 브라우저가 서버에 보내는 요청입니다.
- `Response`: 서버가 돌려주는 결과입니다.
- `API`: 프로그램이 데이터를 주고받는 창구입니다.
- `JSON`: 데이터를 주고받을 때 자주 쓰는 형식입니다.

## 실습 순서

1. API 주소를 확인합니다.
2. 버튼을 누르면 어떤 요청이 일어나는지 예상해봅니다.
3. 응답으로 어떤 데이터가 올지 생각해봅니다.
4. 화면에 출력된 결과를 확인합니다.

## 학생 실습 미션

2. API가 필요한 이유를 예시와 함께 적어보세요.
3. JSON이라는 형식이 왜 필요한지 스스로 설명해보세요.

## 자주 하는 실수

- 브라우저와 서버의 역할을 뒤섞는 경우
- API 주소와 일반 웹페이지 주소를 같은 것으로 이해하는 경우
- JSON을 그냥 텍스트 정도로만 생각하는 경우

## 한 줄 정리

HTTP는 브라우저와 서버가 데이터를 주고받는 규칙이며, API는 그 대화의 창구입니다.
